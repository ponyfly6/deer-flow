**DeerFlow** (**D**eep **E**xploration and **E**fficient **R**esearch **Flow**) is a community-driven deep research framework that seamlessly integrates large language models with specialized tools for web search, content crawling, and Python code execution. Built on the shoulders of the open-source community, DeerFlow empowers users to conduct comprehensive automated research and generate high-quality reports.

DeerFlow is now available in the [Volcengine FaaS Application Center](https://console.volcengine.com/vefaas/region:vefaas+cn-beijing/market). Experience it online via the [demo link](https://console.volcengine.com/vefaas/region:vefaas+cn-beijing/market/deerflow/?channel=github&source=deerflow), or deploy it locally with one-click installation.

Visit our [official website](https://deerflow.tech/) for more details.

## 🚀 Quick Start

DeerFlow is built with Python and includes a modern Web UI powered by Node.js. Follow the steps below to get started.

### Prerequisites

| Tool | Purpose | Installation |
|------|---------|--------------|
| **[uv](https://docs.astral.sh/uv/getting-started/installation/)** | Python environment & dependency management | Automatically handles venv creation |
| **[nvm](https://github.com/nvm-sh/nvm)** | Node.js version management | Recommended for Web UI |
| **[pnpm](https://pnpm.io/installation)** | Node.js package manager | Required for Web UI dependencies |

**System Requirements:**
- Python 3.12+
- Node.js 22+

### Installation

```bash
# Clone the repository
git clone https://github.com/bytedance/deer-flow.git
cd deer-flow

# Install Python dependencies (uv handles venv and packages automatically)
uv sync

# Configure environment variables
cp .env.example .env
# Edit .env with your API keys:
#   - Tavily: https://app.tavily.com/home
#   - Brave Search: https://brave.com/search/api/
#   - TTS credentials (optional)

# Configure LLM settings
cp conf.yaml.example conf.yaml
# See docs/configuration_guide.md for detailed configuration options

# Install Marp for PPT generation (optional)
# https://github.com/marp-team/marp-cli
brew install marp-cli
```

### Web UI Setup (Optional)

```bash
cd web
pnpm install
```

### Running the Application

**Console UI** (quickest way):

```bash
uv run main.py
```

**Web UI** (recommended for full experience):

```bash
# macOS/Linux
./bootstrap.sh -d

# Windows
bootstrap.bat -d
```

> **Note:** By default, the backend binds to `127.0.0.1` for security. To allow external connections, modify the host to `0.0.0.0` in the bootstrap script: `uv run server.py --host 0.0.0.0`. Ensure proper security measures before exposing to external networks.

Access the Web UI at [http://localhost:3000](http://localhost:3000).

For detailed configuration, see the [Configuration Guide](docs/configuration_guide.md).

## 🌟 Features

### 🤖 LLM Integration

- **Universal Model Support**: Integrates with most LLM providers via [LiteLLM](https://docs.litellm.ai/docs/providers)
- **Open Source Models**: Full support for Qwen and other open-source models
- **OpenAI-Compatible API**: Seamless integration with existing tooling
- **Multi-Tier System**: Optimized model selection for different task complexities

### 🔍 Search & Retrieval

| Engine | Type | API Key Required |
|--------|------|------------------|
| [Tavily](https://app.tavily.com/home) | AI-optimized search | Yes |
| [Brave Search](https://brave.com/search/api/) | Privacy-focused | Yes |
| DuckDuckGo | Privacy-focused | No |
| Arxiv | Academic papers | No |
| Searx/SearxNG | Self-hosted metasearch | Host URL required |

Configure your preferred engine in `.env`:

```bash
SEARCH_API=tavily  # Options: tavily, duckduckgo, brave_search, arxiv
```

### 📃 RAG Integration

Connect to your private knowledge base:

- **[RAGFlow](https://github.com/infiniflow/ragflow)**: Open-source RAG engine with document mention support

```bash
# .env configuration
RAG_PROVIDER=ragflow
RAGFLOW_API_URL="http://localhost:9388"
RAGFLOW_API_KEY="ragflow-xxx"
RAGFLOW_RETRIEVAL_SIZE=10
RAGFLOW_CROSS_LANGUAGES=English,Chinese,Spanish,French,German,Japanese,Korean
```

### 🔗 MCP Integration

Extend DeerFlow's capabilities through Model Context Protocol:

- Private domain data access
- Knowledge graph integration
- Web browsing and automation
- Custom tool development

### 💬 Human-in-the-Loop

- **Intelligent Clarification**: Multi-turn dialogue to refine vague research topics
- **Plan Review**: Review and modify research plans before execution
- **Natural Language Editing**: Modify plans using conversational feedback
- **Auto-Accept Mode**: Skip review for fully automated workflows

### 📝 Report Post-Editing

- Notion-style block editing powered by [Tiptap](https://tiptap.dev/)
- AI-assisted refinement: polishing, summarization, expansion
- Rich media support with embedded images

### 🎙️ Content Generation

- **Podcast Creation**: AI-powered script generation and audio synthesis
- **Presentation Slides**: Automated PowerPoint generation with customizable templates

## 🏗️ Architecture

DeerFlow implements a modular multi-agent architecture built on [LangGraph](https://github.com/langchain-ai/langgraph), enabling flexible state-based workflows with well-defined message passing.

![Architecture Diagram](./assets/architecture.png)

> [View live architecture demo](https://deerflow.tech/#multi-agent-architecture)

### Core Components

| Component | Role | Responsibilities |
|-----------|------|------------------|
| **Coordinator** | Entry Point | Workflow lifecycle management, user interface |
| **Planner** | Strategist | Task decomposition, execution planning, flow control |
| **Researcher** | Executor | Web search, information gathering, MCP service calls |
| **Coder** | Analyst | Code execution, data analysis, technical tasks |
| **Reporter** | Synthesizer | Finding aggregation, report generation |

### Workflow

1. **Coordinator** receives user input and initiates the research process
2. **Planner** analyzes objectives and creates structured execution plans
3. **Research Team** (Researcher + Coder) executes the plan using specialized tools
4. **Reporter** aggregates findings and generates comprehensive reports

```bash
    "speed_ratio": 1.0,
    "volume_ratio": 1.0,
    "pitch_ratio": 1.0
}' \
--output speech.mp3
```

### Debugging with LangGraph Studio

DeerFlow includes built-in support for [LangGraph Studio](https://github.com/langchain-ai/langgraph) debugging.

**macOS:**

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
uvx --refresh --from "langgraph-cli[inmem]" --with-editable . --python 3.12 langgraph dev --allow-blocking
```

**Windows/Linux:**

```bash
pip install -e .
pip install -U "langgraph-cli[inmem]"
langgraph dev
```

After starting, access:
- **API**: http://127.0.0.1:2024
- **Studio UI**: https://smith.langchain.com/studio/?baseUrl=http://127.0.0.1:2024
- **API Docs**: http://127.0.0.1:2024/docs

### LangSmith Tracing

Enable detailed workflow tracing:

```bash
# .env configuration
LANGSMITH_TRACING=true
LANGSMITH_ENDPOINT="https://api.smith.langchain.com"
LANGSMITH_API_KEY="your-api-key"
LANGSMITH_PROJECT="your-project-name"
```

### Checkpointing

DeerFlow supports persistent workflow state via MongoDB or PostgreSQL:

```bash
# .env configuration
LANGGRAPH_CHECKPOINT_SAVER=true
LANGGRAPH_CHECKPOINT_DB_URL="mongodb://localhost:27017/"
# LANGGRAPH_CHECKPOINT_DB_URL="postgresql://localhost:5432/postgres"
```

> **Note:** For PostgreSQL, use `langgraph-checkpoint-postgres==2.0.21` due to [issue #5557](https://github.com/langchain-ai/langgraph/issues/5557).

## 🐳 Docker

### Single Container

```bash
# Build image
docker build -t deer-flow-api .

# Run container
docker run -d -t -p 127.0.0.1:8000:8000 --env-file .env --name deer-flow-api-app deer-flow-api
```

```bash
docker compose build
docker compose up
```

### Command Line Usage

```bash
# Direct query
uv run main.py "What factors influence AI adoption in healthcare?"

# Custom planning parameters
uv run main.py --max_plan_iterations 3 "How does quantum computing impact cryptography?"

# Interactive mode
uv run main.py --interactive

# View all options
uv run main.py --help
```

### Human-in-the-Loop Workflow

1. **Plan Review**: System presents the research plan for approval
2. **Feedback Options**:
   - Accept: `[ACCEPTED]`
   - Modify: `[EDIT PLAN] Add more steps about technical implementation`
3. **Auto-Accept**: Set `auto_accepted_plan: true` in API requests

### API Integration

```json
{
  "messages": [{ "role": "user", "content": "What is quantum computing?" }],
  "thread_id": "my_thread_id",
  "auto_accepted_plan": false,
  "feedback": "[EDIT PLAN] Include more about quantum algorithms"
}
```
